# ITDS301_Project 3

**โปรเจกนี้คือการจำลองระบบแลกเปลี่ยนข้อมูลระหว่างโรงพยาบาล (HIS ↔ HIS) โดยใช้มาตรฐาน FHIR แทนการส่งข้อมูลผ่าน database ตรง ๆ**

---

## 🔑 แนวคิดหลัก

* เราจะสร้าง **เว็บสำหรับกรอกข้อมูลคนไข้**
* แล้วระบบจะ:
  1. แปลงข้อมูลเป็น **FHIR Resource**
  2. รวมข้อมูลเป็น **Bundle**
  3. ส่งผ่าน **API ไปยัง HAPI FHIR Server**

👉 เป้าหมายคือให้ระบบ “คุยกันด้วยมาตรฐาน FHIR”

---

## 📦 ข้อมูลที่ต้องมี (อย่างน้อย 3 ตัว)
* `Patient` → ข้อมูลคนไข้
* `Encounter` → มาวันไหน / มารักษาอะไร
* `MedicationRequest` → ใบสั่งยา

---

## 🔁 Flow การทำงาน

```id="3mv6nt"
ผู้ใช้กรอกข้อมูล → ระบบแปลงเป็น FHIR → รวมเป็น Bundle → ส่ง API → HAPI Server รับข้อมูล
```

---

## ⚙️ สิ่งที่ระบบต้องมี
* หน้าเว็บกรอกข้อมูล (Form)
* Backend สำหรับแปลงเป็น FHIR
* การส่ง API (POST)
* HAPI FHIR Server (รันด้วย Docker)
* ทดสอบให้ข้อมูลเข้า server ได้จริง
---

## ❗ สิ่งสำคัญที่อาจารย์เน้น
* ❌ ห้ามส่งข้อมูลผ่าน database โดยตรง
* ✅ ต้องส่งผ่าน FHIR API
* ✅ ต้องใช้ FHIR JSON ตามโครงสร้างจริง
* ✅ ต้องมีอย่างน้อย 3 resource
* ✅ ต้องมีความสัมพันธ์ระหว่าง resource
